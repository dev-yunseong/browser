# 리스트, 테이블 및 대화형 입력창 고도화 계획

브라우저의 표현력을 높이기 위해 복잡한 구조의 데이터(List, Table)를 지원하고, 사용자가 직접 텍스트를 입력할 수 있는 대화형 입력창(Input) 기능을 구현합니다.

## 1. 리스트 및 테이블 지원 (List & Table)
- **리스트 (`<ul>`, `<li>`):**
    - `<li>` 요소를 위한 불렛(Bullet) 포인트 렌더링 기능을 `render.rs`에 추가합니다.
    - 리스트 아이템의 들여쓰기(Indentation)를 레이아웃 엔진에서 처리합니다.
- **테이블 (`<table>`, `<tr>`, `<td>`):**
    - 단순화된 테이블 레이아웃 모델을 도입합니다.
    - `<tr>`은 가로 행을 형성하고, `<td>`는 해당 행의 너비를 균등하게 분할하여 차지하도록 설계합니다.
    - 테이블 경계선을 가시화하기 위한 테두리 렌더링을 강화합니다.

## 2. 대화형 텍스트 입력창 (Interactive Text Edit)
- **대상 요소:** `<input type="text">`, `<textarea>`
- **작동 원리 (Overlay Strategy):**
    - 레이아웃 엔진이 입력창의 위치와 크기를 계산하여 `FormControl` 리스트에 저장합니다.
    - `egui` 인터페이스 루프에서 이 좌표 정보를 바탕으로 **실제 egui 텍스트 위젯**을 해당 위치에 겹쳐서(Overlay) 표시합니다.
    - 이를 통해 폰트 입력, 커서 관리, 복사/붙여넣기 등 복잡한 입력 로직을 네이티브 수준으로 제공합니다.

## 3. 구현 단계
1. **`layout.rs`:** 테이블 및 입력창 전용 레이아웃 로직 추가 및 `FormControl` 정보 수집 기능 구현.
2. **`render.rs`:** 리스트 불렛 및 테이블 구조 시각화 로직 추가.
3. **`main.rs`:** `BrowserApp` 상태에 입력창 데이터를 유지하고, 화면 업데이트 시 `egui` 위젯을 렌더링 영역 위에 배치.

---

## 4. 검증
- 한글 입력이 가능한 입력창이 웹페이지 내에 나타나고 실제로 타이핑이 되는지 확인합니다.
- 테이블 내의 텍스트들이 가로/세로 칸에 맞춰 정렬되는지 확인합니다.
