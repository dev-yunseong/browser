# 브라우저 고도화 기획안 (추상적 기획)

현재의 정적 HTML/CSS 렌더러 프로토타입을 실제 사용 가능한 **대화형 데스크톱 애플리케이션**으로 진화시키기 위한 단계별 고도화 계획입니다.

## 1. 프로젝트 목표 (Vision)
*   **정적 렌더러에서 동적 브라우저로:** 단순히 이미지를 저장하는 도구가 아닌, 실시간으로 페이지를 탐색하고 상호작용할 수 있는 윈도우 기반 애플리케이션 구축.
*   **사용자 인터페이스(UI) 탑재:** 주소창, 뒤로 가기, 앞으로 가기, 새로고침 등 브라우저 핵심 UI(Chrome) 제공.
*   **인터랙티브 웹 콘텐츠:** 클릭 가능한 링크, 입력 가능한 폼(Form) 등 기본적인 사용자 이벤트 처리.

## 2. 주요 개발 단계 (High-level Phases)
*   **Phase A: GUI 환경 통합**
    *   `winit` 및 `softbuffer` 또는 `egui`를 도입하여 실제 OS 윈도우 생성.
    *   렌더링 결과를 파일(`output.png`)이 아닌 윈도우 프레임 버퍼로 실시간 출력.
*   **Phase B: 브라우저 UI(Chrome) 디자인 및 구현**
    *   상단 영역: 주소 입력창(URL Bar), 제어 버튼(Nav Buttons).
    *   하단 영역: 웹 콘텐츠 렌더링 영역(Viewport).
*   **Phase C: 텍스트 렌더링 고도화**
    *   `ab_glyph` 또는 `vello`를 연동하여 실제 폰트를 화면에 고품질로 출력.
*   **Phase D: 내비게이션 및 이벤트 시스템**
    *   주소창 엔터 입력 시 해당 URL로 재요청 및 리렌더링.
    *   마우스 클릭 이벤트를 레이아웃 박스 좌표와 매칭하여 링크 클릭 처리.
*   **Phase E: 성능 최적화 및 상태 관리**
    *   변경사항이 있을 때만 화면을 다시 그리는 효율적인 렌더링 루프 구축.
