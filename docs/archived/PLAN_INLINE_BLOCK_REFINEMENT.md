# 인라인-블록 레이아웃 시스템 정밀화 계획

요소들이 무조건 수직으로 쌓이지 않고, 인라인 요소(a, span, text 등)들이 가로로 자연스럽게 흐르며 배치되도록 레이아웃 엔진을 고도화합니다.

## 1. 레이아웃 모델 고도화
- **Container Context 도입:** 
    - 부모 요소가 블록 컨테이너인지 인라인 컨테이너인지에 따라 자식들의 배치 전략을 변경합니다.
    - 인라인 요소들은 가로 좌표(`current_x`)를 공유하며, 가용 너비를 초과할 때만 수직 좌표(`current_y`)를 갱신합니다.
- **라인 브레이킹(Line Breaking) 개선:**
    - 인라인 요소 배치 중 `current_x + element_width > parent_width`인 경우, 자동으로 `current_x = start_x`로 초기화하고 `current_y`를 한 줄 높이만큼 내립니다.

## 2. 텍스트 및 인라인 요소 연동
- **복합 인라인 배치:** 글자(Text)와 링크(a)가 섞여 있을 때, 이들이 하나의 연속된 문장처럼 가로로 이어지도록 수정합니다.
- **높이 계산 정밀화:** 한 줄에 여러 크기의 인라인 요소가 있을 경우, 해당 줄의 최대 높이를 계산하여 다음 줄의 시작 위치를 결정합니다.

## 3. 구현 단계
1.  **`src/layout.rs`:** `build_layout_tree` 내부의 자식 순회 루프에서 `current_x`와 `current_y`를 실시간으로 업데이트하며 줄바꿈 로직을 통합합니다.
2.  **`src/layout.rs`:** `DisplayType` 판별 로직을 강화하여 더 많은 태그를 지원합니다.
3.  **`src/render.rs`:** 레이아웃 엔진에서 계산된 최종 좌표를 바탕으로 픽셀 블렌딩 렌더링을 유지합니다.

---

## 4. 검증
- 메뉴 바(Menu Bar) 형태의 링크 나열이 가로로 한 줄에 나오는지 확인합니다.
- 긴 문장 중간에 링크가 섞여 있어도 줄바꿈이 자연스럽게 일어나는지 확인합니다.
