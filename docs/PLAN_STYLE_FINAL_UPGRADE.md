# 스타일 엔진 최종 고도화 계획

실제 웹사이트의 화려한 색상과 디자인을 그대로 재현하기 위해 외부 리소스 로드 및 정밀한 스타일 매칭 시스템을 구축합니다.

## 1. 외부 CSS 로드 (External Stylesheets)
- **목표:** `<link rel="stylesheet">` 태그를 찾아 외부 CSS 파일의 내용을 비동기로 다운로드합니다.
- **연동:** 다운로드된 모든 CSS 소스를 하나로 합쳐 전체 스타일시트(`Stylesheet`)를 구축합니다.

## 2. 정밀 CSS 선택자 지원 (Class & ID Selectors)
- **파서 확장:** `.class-name`, `#id-name` 형태의 선택자를 토큰화하고 파싱할 수 있도록 개선합니다.
- **매칭 로직 강화:** `build_style_tree`에서 각 노드의 `class`와 `id` 속성을 체크하여 스타일을 적용합니다.
    - 예: `<div class="header">`는 `div` 규칙뿐만 아니라 `.header` 규칙도 적용받습니다.

## 3. 스타일 우선순위 및 상속 정밀화
- **우선순위(Specificity) 기초:** 태그 < 클래스 < 아이디 순으로 스타일이 덮어씌워지도록 간단한 우선순위 로직을 도입합니다.
- **색상 파싱 보강:** 인식하지 못했던 특수 색상 포맷들에 대한 예외 처리를 강화합니다.

---

## 4. 구현 단계
1.  **`src/main.rs`:** 외부 링크 리소스를 수집하고 다운로드하는 파이프라인 추가.
2.  **`src/css.rs`:** 클래스/아이디를 포함한 복합 선택자 파싱 로직 구현.
3.  **`src/style.rs`:** 속성 기반 스타일 매칭 엔진 업그레이드.
