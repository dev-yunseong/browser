# 브라우저 디자인 및 세부 명세 보고서 (Detailed Design)

브라우저 고도화를 위한 아키텍처 및 디자인 세부 사항을 다룹니다.

## 1. 시스템 아키텍처 디자인
전체 시스템을 **Chrome(애플리케이션 UI)**와 **Content(웹 엔진)** 영역으로 이원화하여 관리합니다.

```
+-----------------------------------------------------+
| [ < ] [ > ] [ Refresh ] [ http://example.com      ] |  <- Chrome Area (egui)
+-----------------------------------------------------+
|                                                     |
|                                                     |
|               Viewport Area (Web Content)           |  <- Engine Output
|                                                     |
|                                                     |
+-----------------------------------------------------+
```

### 가. 기술 스택 (Proposed Tech Stack)
*   **GUI 및 이벤트 루프:** `winit` + `egui` (네이티브 수준의 성능과 쉬운 UI 제작)
*   **실시간 렌더링:** `softbuffer` (윈도우 픽셀 버퍼 직접 쓰기) 및 `tiny-skia` (2D 드로잉 최적화)
*   **폰트 엔진:** `ab_glyph`를 통해 트루타입(TTF) 폰트 래스터화 및 화면 출력.

## 2. 세부 컴포넌트 설계

### 가. 상태 관리 (Application State)
*   `BrowserState` 구조체 정의:
    *   `current_url`: 현재 방문 중인 URL.
    *   `history`: 방문 기록 리스트 (뒤로 가기/앞으로 가기 지원용).
    *   `dom_root`: 현재 파싱된 DOM 트리.
    *   `layout_root`: 현재 계산된 레이아웃 트리.

### 나. 입력 및 상호작용 (Event Handling)
1.  **URL 입력:** 주소창에 URL을 입력하고 Enter를 누르면 `reqwest` 비동기 요청을 트리거하여 엔진의 전체 파이프라인(Download -> Parse -> Style -> Layout -> Render)을 재실행합니다.
2.  **클릭 감지:** 윈도우 마우스 이벤트를 캡처하여 Viewport 내부 좌표로 변환한 뒤, Layout Tree의 박스들과 충돌 검사(Hit Test)를 수행하여 `<a>` 태그 등 클릭 이벤트를 감지합니다.

### 다. 실시간 렌더링 파이프라인
*   **렌더링 갱신 주기:** 페이지가 로드되거나, 창 크기가 조절되거나, 상호작용이 있을 때만 Dirty Flag를 설정하여 불필요한 연산을 방지합니다.
*   **픽셀 스트리밍:** `tiny-skia`로 생성된 `Pixmap` 데이터를 `softbuffer`의 프레임 버퍼로 고속 복사하여 지연 시간(Latency)을 최소화합니다.

## 3. UI/UX 디자인 가이드라인
*   **미니멀리즘:** 주소창은 둥근 모서리를 적용하고, 로딩 중에는 애니메이션이나 상태 메시지를 표시합니다.
*   **반응형 레이아웃:** 윈도우 창 크기가 조절되면 레이아웃 엔진이 즉각적으로 뷰포트 너비를 다시 계산(Re-layout)하여 화면을 갱신합니다.
