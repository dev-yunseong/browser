# 레이아웃 및 시각 요소 고도화 계획

브라우저의 기본 접속 페이지를 변경하고, 요소들이 가로로 배치될 수 있는 플로우 레이아웃(Flow Layout)과 색상 표현 기능을 강화합니다.

## 1. 기본 설정 및 UI 업데이트
- **기본 URL 변경:** 앱 실행 시 로드되는 기본 주소를 `https://yunseong.dev`로 수정합니다.
- **배경색/글자색 고도화:** CSS에서 정의된 `color`와 `background-color`가 모든 요소에 정확히 적용되도록 렌더링 엔진을 개선합니다.

## 2. 플로우 레이아웃(Flow Layout) 도입
- **수평 배치 지원:** 현재 모든 요소를 위에서 아래로만 쌓는 방식에서 벗어나, 인라인 성격의 요소들은 가로로 이어서 배치합니다.
- **인라인/블록 모델 구분:**
    - **Block (div, p, h1 등):** 항상 새로운 줄에서 시작하고 전체 너비를 차지합니다.
    - **Inline (a, span 등):** 가로 공간이 허용하는 한 옆으로 나란히 배치하며, 공간이 부족할 때만 줄바꿈을 수행합니다.
- **좌표 계산 로직 개선:** `build_layout_tree` 함수가 `current_x`와 `current_y`를 지능적으로 관리하여 줄바꿈 지점을 계산하도록 수정합니다.

## 3. 구현 단계
1.  **`src/main.rs`:** 기본 URL 상수를 수정합니다.
2.  **`src/layout.rs`:** 요소의 태그에 따라 인라인/블록 여부를 판단하고, 가로 위치를 추적하며 레이아웃을 잡는 로직을 구현합니다.
3.  **`src/render.rs`:** 텍스트 및 박스 렌더링 시 색상 속성을 최우선으로 반영하도록 수정합니다.

## 4. 검증 및 테스트
- `yunseong.dev` 페이지를 로드했을 때, 메뉴 항목이나 링크들이 가로로 정렬되는지 확인합니다.
- 텍스트와 배경색이 의도한 대로 채워지는지 시각적으로 검증합니다.
