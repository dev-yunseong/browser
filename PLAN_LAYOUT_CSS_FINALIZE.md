# CSS 및 레이아웃 엔진 최종 고도화 계획

브라우저의 시각적 완성도를 실제 웹 브라우저 수준으로 끌어올리기 위해 텍스트 줄바꿈(Text Wrapping) 및 정밀한 CSS 렌더링 시스템을 구축합니다.

## 1. 텍스트 줄바꿈 (Text Wrapping) 구현
*   **문제점:** 현재 모든 텍스트가 한 줄로만 처리되어 부모 박스 너비를 초과하면 잘리거나 튀어나감.
*   **해결책:** 
    *   부모 박스의 가용 너비(Available Width)를 기준으로 텍스트를 단어 단위로 쪼개어 여러 줄로 나누는 로직 추가.
    *   각 줄(Line)의 높이를 합산하여 텍스트 노드의 최종 높이를 결정.
    *   줄바꿈 시 `y` 좌표를 자동으로 다음 줄로 넘기는 `Line Breaking` 시스템 구축.

## 2. 정밀한 CSS 속성 반영
*   **테두리(Border) 렌더링 정상화:**
    *   현재 모든 박스에 강제로 그려지는 디버그용 테두리를 제거.
    *   CSS에 `border` 속성이 명시된 경우에만 해당 두께와 색상으로 렌더링.
*   **배경색(Background) 및 글자색(Color):**
    *   상속(Inheritance) 개념을 일부 도입하여 부모의 글자색이 자식에게 전달되도록 개선.
*   **박스 모델 정밀화:** `padding`과 `margin`이 실제 레이아웃 크기에 정확히 합산되도록 로직 재검증.

## 3. 구현 단계 (Phased Approach)
1.  **CSS 파서 고도화:** 복합 속성(Shorthand) 파싱 강화 및 기본값 처리 로직 개선.
2.  **레이아웃 엔진 핵심 수정:** 텍스트 노드 처리 시 가용 너비를 체크하여 `Vec<Line>` 구조로 레이아웃 생성.
3.  **렌더링 엔진 수정:** 강제 테두리 로직 제거 및 줄바꿈된 텍스트 렌더링 지원.

---

## 4. 검증 및 테스트
*   **긴 문장 테스트:** 아주 긴 문장을 로드했을 때 창 너비에 맞춰 자동으로 줄바꿈이 일어나는지 확인.
*   **복합 스타일 테스트:** 배경색만 있는 박스, 테두리만 있는 박스 등이 각각 CSS 정의대로 나오는지 확인.
