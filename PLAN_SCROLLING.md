# 스크롤 기능 고도화 계획

웹페이지의 전체 콘텐츠를 탐색할 수 있도록 수직 스크롤 기능을 구현합니다.

## 1. 콘텐츠 전체 높이 감지
- **문제:** 현재 렌더링 영역이 800x600으로 고정되어 있어 긴 페이지의 아래쪽이 잘림.
- **해결:** 레이아웃 엔진이 계산한 `final_y` 값을 콘텐츠의 전체 높이로 사용합니다.

## 2. 동적 픽셀 버퍼(Pixmap) 생성
- 고정된 600px 높이가 아닌, 레이아웃 계산 결과에 따른 가변 높이로 `tiny-skia::Pixmap`을 생성합니다.
- 매우 긴 페이지의 경우 메모리 효율을 위해 최대 높이 제한을 두거나 분할 렌더링을 고려할 수 있으나, 초기에는 전체 높이 렌더링을 시도합니다.

## 3. GUI 스크롤 영역 통합
- `egui::ScrollArea`를 활용하여 생성된 대형 텍스처를 감쌉니다.
- `egui`가 제공하는 네이티브 스크롤바를 통해 사용자가 마우스 휠이나 드래그로 페이지를 이동할 수 있게 합니다.

## 4. 좌표 매핑 보정
- 스크롤 위치에 관계없이 링크 클릭(Hit Testing)이 정확한 위치에서 동작하도록 `egui`의 좌표계 변환을 활용합니다.

---

## 5. 구현 단계
1.  **`src/main.rs`:** `fetch_and_process` 함수에서 레이아웃 결과로 받은 `final_y`를 바탕으로 `Pixmap` 크기를 결정하도록 수정합니다.
2.  **`src/main.rs`:** UI 업데이트 루프에서 `ScrollArea` 설정을 점검하여 대형 이미지가 정상적으로 스크롤되도록 보장합니다.
