# 상호작용 및 내비게이션 정밀화 계획

브라우저의 클릭 상호작용이 정확하게 동작하도록 좌표 시스템을 보정하고, 링크의 시각적 요소를 강화합니다.

## 1. 링크 시각화 (Link Styling)
- **목표:** 사용자가 어디를 클릭해야 할지 명확히 알 수 있도록 스타일 적용.
- **수정:** `render.rs`에서 `<a>` 태그의 자식 텍스트 노드인 경우 파란색(#0000EE)과 밑줄을 강제로 렌더링하도록 로직 추가.

## 2. 스크롤 대응 좌표 보정 (Coordinate Correction)
- **문제:** 스크롤 시 화면 클릭 좌표와 레이아웃 박스 좌표가 일치하지 않음.
- **해결:** `egui`의 위젯 좌표계에서 상대 좌표를 추출할 때, 스크롤 영역의 오프셋이 이미 반영된 `response.interact_pointer_pos()`를 더 신뢰성 있게 활용.

## 3. 인라인 레이아웃 높이 전파 수정
- **문제:** 인라인 박스(`<a>`)가 내부 텍스트의 높이를 부모에게 전달하지 않아 클릭 영역이 좁게 형성됨.
- **해결:** `layout.rs`에서 인라인 박스의 최종 높이가 내부 콘텐츠의 최대 높이를 반영하도록 수정.

---

## 4. 구현 단계
1.  **`src/render.rs`:** 링크 스타일(파란색 + 밑줄) 및 텍스트 블렌딩 로직 보강.
2.  **`src/layout.rs`:** 인라인 요소의 높이 및 좌표 전달 로직 정교화.
3.  **`src/main.rs`:** 이미지 클릭 시 `egui::Sense::click()`을 명시적으로 활용하여 정확도 향상.
