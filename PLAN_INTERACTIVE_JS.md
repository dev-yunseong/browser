# 상호작용 및 JavaScript 엔진 통합 계획

브라우저의 레이아웃 유연성을 높이고, 클릭 상호작용을 완성하며, 현대적인 브라우저의 필수 요소인 JavaScript 엔진 도입을 위한 로드맵을 수립합니다.

## 1. 가로 정렬 및 유연한 레이아웃 (Side-by-side Blocks)
- **`display: inline-block` 지원:** 
    - 태그에 상관없이 CSS `display` 속성에 따라 인라인처럼 가로로 흐르되, 블록처럼 너비와 높이를 가질 수 있는 모델을 구현합니다.
- **수평 흐름 알고리즘 개선:**
    - 현재 줄의 남은 가용 너비를 체크하여 요소가 들어갈 수 있으면 옆에 붙이고, 부족하면 줄바꿈(Line Break)을 수행하는 로직을 정교화합니다.

## 2. 인터랙티브 내비게이션 완성
- **정밀한 히트 테스트 (Hit Testing):**
    - 가로/세로로 복잡하게 얽힌 레이아웃 트리에서 마우스 클릭 좌표를 정확히 추적하여 `<a>` 태그의 `href`를 추출합니다.
- **URL 이동 시스템:**
    - 클릭된 URL이 상대 경로인 경우 현재 기본 URL과 결합하여 절대 경로로 변환 후 새 페이지를 로드합니다.

## 3. JavaScript 엔진 통합 검토 및 계획
- **기술적 검토 (Feasibility Study):**
    - **후보 1: Boa (Pure Rust):** Rust로만 작성되어 통합이 매우 쉽고 안전하지만, 최신 JS 표준 지원과 성능이 다소 낮을 수 있음. (추천)
    - **후보 2: QuickJS (via rquickjs):** 매우 가볍고 빠르며 호환성이 좋지만 C 라이브러리 바인딩이 필요함.
    - **후보 3: rusty_v8:** 구글 V8 엔진 사용. 성능은 최고이나 빌드 시간이 매우 길고 바이너리가 무거움.
- **통합 단계 (JS Roadmap):**
    1. **엔진 임베딩:** `Boa` 크레이트를 프로젝트에 추가.
    2. **DOM 브릿지 구축:** Rust의 DOM 노드를 JS에서 접근하고 수정할 수 있는 바인딩 객체 생성.
    3. **글로벌 객체 제공:** `window`, `document`, `console.log` 등 기본 API 구현.
    4. **스크립트 실행:** `<script>` 태그 내부의 코드를 파싱 시점에 실행하거나 외부 `.js` 파일을 로드.

---

## 4. 구현 단계
1. **Phase 1:** `layout.rs` 수정하여 가로 정렬(`inline-block`) 로직 완성.
2. **Phase 2:** 마우스 상호작용 및 클릭 내비게이션 버그 수정.
3. **Phase 3:** JavaScript 엔진(`Boa`) 도입을 위한 프로토타입 작성.
