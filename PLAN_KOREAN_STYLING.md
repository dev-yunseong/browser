# 한글 렌더링 및 스타일링 고도화 계획

한글을 완벽하게 지원하고, CSS 스타일(크기, 색상, 배경)이 계층적으로 올바르게 적용되도록 브라우저 엔진을 강화합니다.

## 1. 한글 렌더링 지원 (Korean Support)
- **문제:** 현재 사용 중인 `DejaVuSans` 폰트는 한글 글리프를 포함하지 않아 한글이 깨지거나 표시되지 않음.
- **해결:** 한글을 지원하는 오픈소스 폰트(나눔고딕 등)를 프로젝트에 내장하고 렌더링 엔진에서 이를 사용하도록 변경합니다.

## 2. CSS 스타일 상속 및 적용 강화
- **스타일 상속 (Inheritance):** 
    - `color`, `font-size` 등은 부모 요소의 값을 자식이 명시적으로 재정의하지 않으면 물려받도록 `style.rs` 로직을 개선합니다.
- **배경색 적용:** 모든 레이아웃 박스에 대해 `background-color`가 존재할 경우 정확한 영역에 색을 칠합니다.
- **글자 크기 동적 반영:** `font-size` 속성을 `ab_glyph`의 스케일 계산에 실시간으로 반영합니다.

## 3. 구현 단계
1.  **폰트 교체:** `assets/fonts/NanumGothic.ttf`를 다운로드하여 기존 `DejaVuSans.ttf`를 대체합니다.
2.  **`style.rs` 수정:** `build_style_tree`에서 부모의 스타일 맵을 참조하여 상속 가능한 속성들을 전파합니다.
3.  **`render.rs` 수정:** 텍스트 렌더링 시 색상과 크기 정보를 스타일 트리에서 정확히 추출하여 반영합니다.

---

## 4. 검증
- `yunseong.dev` 및 한글이 포함된 페이지를 로드하여 한글이 올바르게 출력되는지 확인합니다.
- 부모 `div`에 지정된 빨간색 글자색이 내부 `p` 태그의 글자들에게도 적용되는지 시각적으로 확인합니다.
