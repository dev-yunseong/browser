# 브라우저 안정성 및 하드웨어 한계 대응 계획

비정상적으로 큰 페이지 높이 계산으로 인한 앱 패닉 문제를 해결하고, 렌더링 성능을 안정화합니다.

## 1. 레이아웃 높이 계산 버그 수정
- **문제:** 자식 노드의 높이가 부모에게 합산될 때 중복으로 계산되어 기하급수적으로 높이가 늘어남.
- **해결:** `max_y_in_box`를 계산할 때 자식 박스의 실제 하단 좌표(`y + height`)만을 기준으로 삼도록 로직을 단순화합니다.

## 2. 하드웨어 가속 텍스처 한계 대응
- **제한:** 대부분의 GPU 및 GUI 프레임워크(`egui`)는 단일 텍스처 크기를 16,384px(또는 8,192px)로 제한함.
- **해결:** 
    - 렌더링 전 최종 높이를 체크하여 시스템 한계치(16,384px)를 넘지 않도록 강제 제한(Cap)을 겁니다.
    - 초과하는 부분은 현재 단계에서는 렌더링하지 않거나, 추후 타일링(Tiling) 기술로 해결할 기반을 마련합니다.

## 3. 구현 단계
1.  **`src/layout.rs`:** `max_y_in_box` 및 `next_y` 관리 로직 수정.
2.  **`src/main.rs`:** `Pixmap` 생성 전 높이 값을 `16384`로 `clamp` 처리.
