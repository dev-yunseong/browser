# 내비게이션 및 클릭 상호작용 근본 해결 계획

브라우저 내의 링크 클릭이 한 번에 정확히 작동하도록 좌표 시스템을 보정하고, 엔진의 링크 인식 로직을 개선합니다.

## 1. 정밀한 히트 테스트(Hit Testing) 구현
- **문제:** 현재 링크의 클릭 영역이 텍스트 크기를 완벽하게 감싸지 못하거나, 스크롤 시 좌표가 어긋남.
- **해결:** 
    - `layout.rs`에서 `<a>` 태그의 자식 요소(텍스트 등)들이 차지하는 전체 영역을 부모인 `<a>` 박스가 완벽히 포함하도록 크기 계산 로직을 보강합니다.
    - `main.rs`에서 `egui`의 스크롤 오프셋을 100% 반영한 절대 좌표 변환 로직을 적용합니다.

## 2. 시각적 피드백 및 디버깅 강화
- **마우스 커서:** 링크 위에 마우스를 올렸을 때 커서가 손 모양(`PointingHand`)으로 변하는 것을 확실히 보장합니다.
- **상태 로깅:** 클릭 시도 시 터미널 콘솔에 `[Click Attempt] x: ..., y: ...`와 같은 좌표 로그와 검색된 링크 정보를 출력하여 오동작 시 즉시 원인을 파악할 수 있게 합니다.
- **링크 시각화:** `render.rs`에서 링크 텍스트의 파란색 처리와 밑줄이 누락되지 않도록 로직을 수정합니다.

## 3. 구현 단계
1.  **`src/layout.rs`:** 인라인 박스(`<a>`)의 크기 결정 로직 및 자식 영역 합산 로직 수정.
2.  **`src/render.rs`:** 링크 색상 적용 로직의 버그 수정 (부모 태그 확인 절차 강화).
3.  **`src/main.rs`:** 클릭 감지 로직 단순화 및 좌표 로깅 추가.

---

## 4. 검증
- `cargo run` 실행 후 터미널 로그를 통해 링크가 몇 개 로드되었는지 확인합니다.
- 실제 클릭 시 터미널에 내비게이션 로그가 찍히며 페이지가 전환되는지 확인합니다.
